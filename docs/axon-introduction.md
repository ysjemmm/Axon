# Axon —— 我自己做的一个 AI IDE，分享给大家

## 先说几句大实话

这不是某个团队立项做的产品，是我一个人在业余时间，从 Code OSS 这个开源底座上一点点改出来的。

我自己每天用着写代码，越用越觉得"光给同事用真的太可惜了"，所以整理了这篇介绍，想把它分享给团队里的大家。

正因为是一个人维护，所以有什么问题、想要什么功能，**直接找我（码扬）就行**，不用走流程，我改起来也快。这大概是「一个人做的工具」相对于「大厂产品」唯一的优势了——响应快，离用户近。

---

## 为什么会有 Axon？

我自己用过市面上几乎所有主流的 AI 编程工具，体感上它们对我们公司这种场景都不太合适：

| 用起来的真实感受 | 具体问题 |
|------|------|
| **代码不敢随便喂出去** | 很多工具的请求是直接打到境外服务器的，涉及到核心代码、客户数据的项目，谁也不敢用 |
| **配置太麻烦** | 申请 Key、配代理、改配置文件……同事们的精力应该花在写代码上，不是花在"怎么把 AI 装起来" |
| **绑死一个模型** | 只能用工具自带的那一两个模型，没法按任务选最合适的那个 |
| **网络说翻车就翻车** | 访问境外服务一卡就是半天，调试体验直接被打断 |
| **没法接公司自己的东西** | 内部规范、内部系统，工具完全不知道，AI 给的建议经常文不对题 |

**所以我没打算"凑合用一个现成的"，而是干脆自己改了一个——把这几个问题一次性焊死在底层。**

## Axon 到底解决了什么

### 🔐 代码不出公司

所有请求都走公司自己的中转站，不会直接打到外面去。谁在什么时候问了什么、AI 回了什么，都留有记录、可以追溯。**哪怕是比较敏感的项目，也能放心用。**

### 🌐 装好就能用，不用折腾

Axon 默认接好了公司的中转站，**装完打开，直接就能聊**。不用自己去申请 Key、不用配代理、不用看一堆配置文档——跟装个普通软件没区别。

### 🧠 想用哪个模型，自己选

Axon 背后接的不是一个模型，是一整个模型矩阵：

| 厂商 | 可选模型 |
|------|---------|
| **公司中转站**（内置) | GPT、DeepSeek、GLM |
| **OpenAI** | GPT 系列旗舰与轻量模型 |
| **Anthropic Claude** | Claude Opus、Claude Sonnet 系列 |
| **通义千问** | Qwen-Max、Qwen-Plus |
| **智谱** | GLM 系列 |
| **自定义接入** | 任何兼容 OpenAI 接口协议的模型，自己在设置里填一下就能用 |

写代码用哪个、做代码审查用哪个、随手问个小问题用哪个，自己挑，不用将就。

### 🛠 真的能动手干活，不只是聊天

Axon 的 Agent 内核是直接跑在编辑器进程里的，工具调用是函数调用，不用绕一圈网络请求——**这意味着它读文件、改代码、跑命令，基本是"瞬间"的事，不会让你干等着转圈圈。**

它能做的事情包括：

- ✅ **读写文件** —— "帮我看下这个文件"，它直接定位、读取、改完给你看 diff
- ✅ **搜代码** —— "项目里哪些地方调用了这个函数"，秒出结果
- ✅ **跑命令、管进程** —— 不仅能跑一次性命令（比如跑测试），还能启动、停止常驻进程（比如开发服务器），这是很多同类工具做不到的
- ✅ **查最新资料** —— "这个报错最新的解法是什么"，它会自己上网搜，给你出处

AI 不再是"给个建议你自己抄"，而是**直接把活儿干完，你来验收**。

### 👁️ 不用跟它解释你在干什么

这是我觉得最值得讲的一点：你不用复制代码、不用描述项目结构、不用截图报错信息——**光标停在哪、选中了什么、终端刚才报了什么错，它自己就知道。**

对比一下日常体验：

- ❌ **普通 AI 工具**：先复制代码、再解释背景、再贴报错……交代清楚就得几分钟
- ✅ **Axon**：光标停在出错的那一行，直接问"这怎么错的"，它秒懂

**让 AI 自己去看现场，而不是你去喂它——这个体验差距用过就回不去了。**

### 🌐 前端改完，AI 自己帮你点开看

前端开发最烦的是什么？改完代码切到浏览器手动点一遍，没问题再切回来，来回切来切去。

Axon 内置了一个真实的浏览器（Playwright 驱动），可以让 AI **代替你去操作页面**：

- 🚀 写完代码，AI 自己打开浏览器看效果
- 👁️ 截图确认布局、颜色、内容对不对
- 🖱️ 自动填表单、点按钮、模拟真实操作流程
- 📋 自动抓控制台报错、网络请求失败，不用自己开 F12
- 📱 手机、平板、电脑视图一键切换

**举个真实的使用场景：**

```
我："帮我把登录页加上表单校验，然后看看效果对不对"
     ↓
AI 写完代码 → 自己打开浏览器 → 故意输错邮箱 → 点提交
   → 截图看提示有没有弹出来 → 换成正确信息再提交一次
   → 检查控制台有没有报错、接口通不通
   → 回我："弄完了，校验逻辑没问题，接口也通了 ✅"
```

整个过程我都没离开编辑器，干等着看结果就行。

## 复杂任务怎么保证不跑偏：Relay 工作流

让 AI 改一行代码很简单，但让它独立完成一个完整功能，经常做着做着就"理解偏了"。

Axon 的 **Relay 工作流**专门解决这个问题，把一个大任务拆成五步，每一步都有确认点，不会让 AI 自己闷头往前冲：

```
搞清楚需求 → 设计方案 → 拆解任务 → 逐项实现 → 交付验证
```

1. **搞清楚需求** —— 不会上来就写代码，先跟你反复确认边界和验收标准
2. **设计方案** —— 代码一行没写之前，先把思路摆出来给你看
3. **拆解任务** —— 拆成一个个几分钟就能做完的小任务，清清楚楚
4. **逐项实现** —— 一项一项做，每做完一项自动检查质量，**不过关不准做下一项**
5. **交付验证** —— 全部完成、验证通过，可以直接用

**小事直接让它干，大事走 Relay——质量靠流程兜底，不是靠"相信 AI 这次状态好"。**

## 能力还能继续长：三层扩展体系

Axon 不是一锤子买卖，它的能力可以通过三层不断往外扩：

#### 📦 技能包（Skill）

内置了几种"做事方法"，AI 遇到对应任务会自动调用：系统化调试、测试驱动开发、提交前自检、陌生项目快速摸底。团队也可以把自己的规范沉淀成技能，让它越用越懂团队的习惯。

#### 🔌 能力包（Power）

一键接入公司已有系统：查生产日志、看部署状态、搜内部文档、追踪调用链、查数据库——这些原本要自己去好几个系统里翻的事，现在直接问 AI 就行。

#### 🌍 外部接入（MCP）

支持业界标准的 MCP 协议，理论上能接任何你想让 AI 调用的外部系统——项目管理工具、告警系统都行，看团队自己的需要。

**这三层意味着 Axon 的能力边界由用的人决定，不是由我一个人决定。**

## 多任务并行（Beta，先尝鲜）

传统工具一次只能干一件事，Axon 可以让 AI **同时开几个头**：

- **同时调研**：前端怎么做、后端怎么做、数据库怎么设计，三个方向同时派出去查，几秒钟汇总
- **同时实现**：前端页面 + 后端接口 + 测试用例，互不依赖的话可以一起做，总耗时约等于最慢那一项

> ⚠️ 这块还是 Beta，建议先用在互不依赖的独立任务上，有先后顺序的活儿还是老老实实排队来。

[拿来吧你](https://github.com/ysjemmm/Axon-IDE/releases/tag/1.106.0)
